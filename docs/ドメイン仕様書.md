## 1. 概要

### 1.1 目的

29 歳から 65 歳までの家計資産を年単位でシミュレーションし、収入・支出・投資・負債・子ども関連費用の推移を可視化する。日本の主要な税制・社会保険制度を反映した、リアルな資産シミュレーションを提供する。

### 1.2 スコープ

| 項目 | 対応状況 |
|---|---|
| 収入（夫：固定額昇給、妻：昇給なし・産休） | 対応 |
| 収入入力モード（手取り / 額面） | 対応（v2） |
| 所得税（7 段階累進課税 + 復興特別所得税） | 対応（v2） |
| 住民税（所得割 10% + 均等割） | 対応（v2） |
| 社会保険料（健保・厚生年金・雇用・介護） | 対応（v2） |
| 給与所得控除 | 対応（v2） |
| 所得控除（基礎・配偶者・扶養・社保） | 対応（v2） |
| 住宅ローン控除 | 対応（v2） |
| 児童手当（2024 年改正後） | 対応（v2） |
| 固定資産税 | 対応（v2） |
| 退職金（退職所得控除・1/2 課税） | 対応（v2） |
| 投資口座（NISA/課税口座） | 対応（v2） |
| 支出（8 カテゴリ、カテゴリ単位インフレ on/off） | 対応 |
| 投資（複利運用＋年間積立） | 対応 |
| 負債（元利均等方式ローン、複数対応） | 対応 |
| 子ども（複数対応、教育費プリセット＋上書き） | 対応 |
| 年金収入 | 対象外（終了年齢 65 歳のため） |
| ボーナス返済・繰上返済 | 対象外（将来拡張） |
| 複数シナリオの同時比較 | 対象外 |

### 1.3 モデルの前提

| 項目 | 仕様 |
|---|---|
| 時間粒度 | **年単位の離散モデル** |
| 年齢計算 | 期末時点の満年齢で近似（誕生日ベースの日数計算はしない） |
| 税金 | 額面モード: 自動計算 / 手取りモード: 計算なし（後方互換） |
| 通貨 | 日本円（整数、`Math.round` で丸め） |
| 表示 | `Intl.NumberFormat('ja-JP')` で万円単位のカンマ区切り |

## 2. データモデル

### 2.1 シナリオ（Scenario）

シミュレーションの全入力を保持するルートオブジェクト。

| フィールド | 型 | 説明 |
|---|---|---|
| `version` | `1 \| 2` | スキーマバージョン |
| `id` | `string` | 一意識別子 |
| `name` | `string` | シナリオ名 |
| `createdAt` | `string` | 作成日時（ISO 8601） |
| `updatedAt` | `string` | 最終更新日時（ISO 8601） |
| `initialConditions` | `InitialConditions` | 初期条件 |
| `income` | `{ husband, wife }` | 収入設定 |
| `expense` | `{ categories }` | 支出カテゴリ設定 |
| `investment` | `InvestmentConfig` | 投資設定（単一口座、v1 互換） |

| `children` | `Child[]` | 子ども一覧（0 件以上） |
| `taxConfig` | `TaxConfig?` | 税金設定（v2） |

### 2.2 初期条件（InitialConditions）

| フィールド | 型 | 範囲 | 既定値 | 説明 |
|---|---|---|---|---|
| `startYear` | 整数 | 1900〜2200 | 現在年 | シミュレーション開始年 |
| `husbandAge` | 整数 | 0〜120 | 29 | 開始年時点の夫の年齢 |
| `wifeAge` | 整数 | 0〜120 | 29 | 開始年時点の妻の年齢 |
| `initialCashSavings` | 数値 | — | 5,000,000 | 開始時点の預金残高（円） |
| `inflationRatePercent` | 数値 | -20〜20 | 0 | 年率インフレ（%） |
| `endAge` | 整数 | 0〜120 | 65 | シミュレーション終了年齢（夫基準） |

シミュレーション年数 = `endAge - husbandAge + 1`（両端含む）。

### 2.3 収入

#### 2.3.1 夫の収入（HusbandSalary）

| フィールド | 型 | 既定値 | 説明 |
|---|---|---|---|
| `annualTakeHome` | 数値 | 6,000,000 | 初年度の手取り年収（円）— takeHome モード用 |
| `annualGross` | 数値? | 8,000,000 | 初年度の額面年収（円）— gross モード用 |
| `annualRaiseAmount` | 数値 | 100,000 | 毎年の昇給額（円、固定額） |
| `retireYear` | 整数? | — | 退職年（省略時は退職なし） |

**手取りモード**: `年収(i) = annualTakeHome + annualRaiseAmount × i`

**額面モード**: `額面(i) = annualGross + annualRaiseAmount × i` → 税計算パイプラインで手取りを算出

- `retireYear` 以降は収入 0。
- **インフレは給与に適用しない**（実質賃金の目減りを表現する仕様）。

#### 2.3.2 妻の収入（WifeSalary）

| フィールド | 型 | 既定値 | 説明 |
|---|---|---|---|
| `annualTakeHome` | 数値 | 3,000,000 | 手取り年収（円）— takeHome モード用 |
| `annualGross` | 数値? | 4,000,000 | 額面年収（円）— gross モード用 |
| `maternityLeaveYears` | 整数 | 1 | 産休/育休の年数（出産年を含む） |
| `retireYear` | 整数? | — | 退職年 |

- **昇給なし**（要件どおり）。
- `retireYear` 以降は収入 0。

#### 2.3.3 産休ルール

> 出産年を含む `maternityLeaveYears` 年間、妻の収入を 0 とする。

- `maternityLeaveYears = 1` → 出産年のみ 0（要件準拠）
- `maternityLeaveYears = 2` → 出産年＋翌年が 0
- 判定式: `0 <= (year - child.birthYear) < maternityLeaveYears`
- 子ごとに `maternityPauseApplied` フラグがあり、`false` の子は産休に影響しない。
- 複数の子の産休期間が重なった場合でも、妻の収入は 0 のまま（重複による追加ペナルティなし）。

### 2.4 税金・社会保険（v2）

#### 2.4.1 税金設定（TaxConfig）

| フィールド | 型 | 説明 |
|---|---|---|
| `incomeInputMode` | `'takeHome' \| 'gross'` | 収入入力モード |
| `mortgageDeduction` | `MortgageDeductionConfig` | 住宅ローン控除設定 |
| `propertyTax` | `PropertyTaxConfig` | 固定資産税設定 |
| `retirementBonus` | `RetirementBonusConfig` | 退職金設定 |

- `'takeHome'` モード: v1 と完全互換。税計算をスキップし、入力された手取りをそのまま使用。
- `'gross'` モード: 額面年収から社会保険料・所得税・住民税を自動計算して手取りを算出。

**新築住宅の借入限度額:**

| 住宅種別 | 子育て・若者世帯 | その他世帯 |
|---|---|---|
| 認定住宅（長期優良/低炭素） | 5,000 万円 | 4,500 万円 |

**適用順序:**
1. 所得税から控除（基本所得税額が上限）

|---|---|---|

- **所得制限なし**（2024 年 10 月より撤廃）

- シミュレーションでは収入に加算

#### 2.4.10 退職金

- **退職所得控除**:
  - 勤続 20 年以下: 40 万円 × 勤続年数（最低 80 万円）
  - 勤続 20 年超: 800 万円 + 70 万円 ×（勤続年数 − 20 年）
- **課税**: (退職金 − 退職所得控除) × 1/2 に対して累進課税 + 住民税 10%
- 退職年に一括で手取り額を収入に加算

退職金の設定:

| フィールド | 型 | 説明 |
|---|---|---|
| `enabled` | `boolean` | 退職金を含めるか |
| `amount` | 数値 | 退職金額（円） |
| `retireYear` | 整数 | 退職年 |
| `yearsOfService` | 整数 | 勤続年数 |

#### 2.4.11 税計算パイプライン

額面モード時の各年の処理フロー:

```
1. 額面年収（夫婦それぞれ）
2. 社会保険料 = 額面 × 保険料率（年齢で介護保険判定）
3. 給与所得 = 額面 − 給与所得控除
4. 課税所得 = 給与所得 − 所得控除（基礎 + 社保 + 配偶者 + 扶養）
5. 所得税 = 課税所得 × 累進税率 × 1.021（復興特別所得税）
6. 住民税 = 課税所得 × 10% + 5,000
7. 住宅ローン控除 → 所得税から控除 → 残りは住民税から控除
8. 手取り = 額面 − 社保 − 所得税 − 住民税
9. 児童手当を収入に加算
10. 退職金（退職年のみ）を収入に加算
11. 固定資産税を支出に加算
```

### 2.5 支出

#### 2.5.1 支出カテゴリ

全 8 カテゴリが必須。各カテゴリに年額とインフレ適用フラグを設定する。

| キー | 日本語名 | 既定年額 | インフレ既定 |
|---|---|---:|---|
| `fixedCosts` | 固定費 | 1,800,000 | on |
| `carCosts` | 自動車関連 | 400,000 | on |
| `appliances` | 大物家電 | 150,000 | on |
| `food` | 食費 | 720,000 | on |
| `pocketMoney` | お小遣い | 600,000 | off |
| `dailyNecessities` | 日用品 | 240,000 | on |
| `misc` | 雑費 | 200,000 | on |
| `events` | イベント(旅行等) | 300,000 | on |

**計算式**: `当年額 = annualAmount × (applyInflation ? inflationFactor[i] : 1)`

子ども関連費用（食費・雑費・教育費）はここではなく `Child` で管理する。
住宅ローンの返済額も支出カテゴリには含めず、`Loan` の返済額が `totalExpense` に自動加算される。

#### 2.5.2 インフレ

- インフレ係数: `inflationFactor[i] = (1 + inflationRatePercent / 100) ^ i`
- `applyInflation = true` のカテゴリにのみ毎年複利で適用。
- インデックス 0（初年度）は係数 1.0（影響なし）。

#### 2.6.2 教育ステージと年齢対応

| ステージ | 日本語 | 年齢範囲 |
|---|---|---|
| (preSchool) | 未誕生 | age < 0 |
| `daycare` | 保育園 | 0 〜 5 歳 |
| `elementary` | 小学校 | 6 〜 11 歳 |
| `juniorHigh` | 中学校 | 12 〜 14 歳 |
| `highSchool` | 高校 | 15 〜 17 歳 |
| `university` | 大学 | 18 〜 21 歳 |
| (postGrad) | 卒業後 | 22 歳以上 |

- 年齢 = `year - birthYear`（期末満年齢の近似）。
- `preSchool` / `postGrad` では教育費は 0。

#### 2.6.3 教育費の決定順序

```
1. educationCostOverride[stage] が設定済み → その値
2. EDUCATION_COST_PRESET[stage][educationPlan[stage]] → プリセット値
3. ステージ外 (preSchool / postGrad) → 0
```

#### 2.6.4 子ども費用の計算

```
食費 = childFoodAnnual × (applyInflationToChildCosts ? inflationFactor[i] : 1)
雑費 = childMiscAnnual × (同上)
教育費 = 上記の決定順序で取得
合計 = 食費 + 雑費 + 教育費
```

- 子どもが未誕生（age < 0）の年は全項目 0。
- 複数の子がいる場合は全員分を合算。

### 2.8 ローン（負債）

#### 2.8.1 ローンの属性

| フィールド | 型 | 範囲 | 説明 |
|---|---|---|---|
| `id` | `string` | — | 一意識別子 |
| `kind` | `'mortgage' \| 'other'` | — | ローン種別（表示分類用。mortgage は住宅ローン控除の残高計算にも使用） |
| `name` | `string` | — | ローン名 |
| `principal` | 数値 | 0 以上 | 借入元本（円） |
| `annualInterestRatePercent` | 数値 | 0〜50 | 年利（%） |
| `termYears` | 整数 | 0〜60 | 返済年数 |
| `startYear` | 整数 | — | 返済開始年 |

#### 2.8.2 元利均等返済方式

```
月利:    r = annualInterestRatePercent / 100 / 12
総月数:  n = termYears × 12
月返済:  M = P × r / (1 - (1 + r)^(-n))

金利 0% の場合: M = P / n
```

**月次計算**:
```
利息   = 残債 × r
元金   = M - 利息
残債   = 残債 - 元金
```

**年次集約**: 12 ヶ月分の元金・利息・合計を合算。年末の残債を記録。

**最終月の精算**: 最終月は残債を丸ごと返済し、丸め誤差を ±1 円以内に吸収。

**年の判定**:
- `year < startYear` → 返済額 0（ローンは未開始、残債も 0 扱い）
- `startYear <= year < startYear + termYears` → スケジュールに基づく返済
- `year >= startYear + termYears` → 完済済み、返済額 0、残債 0

#### 2.8.3 複数ローン

- ローンは 0 件以上登録可能。
- `loanRepayment` = 全ローンの当年返済額の合計。
- `loanOutstandingBalance` = 全ローンの年末残債の合計。
- `kind === 'mortgage'` のローンの年末残債は住宅ローン控除の計算に使用。

### 2.9 投資

#### 2.9.1 単一口座モード（v1 互換）

| フィールド | 型 | 既定値 | 説明 |
|---|---|---|---|
| `initialBalance` | 数値 | 1,000,000 | 開始時点の投資残高（円） |
| `annualContribution` | 数値 | 600,000 | 年間積立額（円） |
| `annualYieldPercent` | 数値 | 4 | 年利回り（%） |
| `contributionRaiseAmount` | 数値? | — | 積立額の年間増額（円） |
| `stopContributionYear` | 整数? | — | 積立停止年 |

**計算順序**: 期首残高 → 利回り発生 → 積立 → 期末残高

```
利回り   = prevBalance × annualYieldPercent / 100
積立額   = annualContribution + (contributionRaiseAmount ?? 0) × i
           (stopContributionYear 以降は 0)
期末残高 = prevBalance + 利回り + 積立額
```

- **投資積立は「支出」ではなく「預金 → 投資残高への振替」扱い**。
- したがって `totalExpense` には含まれず、`cashSavings` から差し引かれ `investmentBalance` に加算される。
- `investmentAccounts` が存在する場合、単一口座モードは無視され、複数口座モードが使用される。

#### 2.9.2 複数口座モード（v2）

`investmentAccounts[]` が存在する場合に有効。各口座に以下を設定:

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | `string` | 一意識別子 |
| `name` | `string` | 口座名 |
| `accountType` | `'nisa' \| 'taxable'` | 口座種別 |
| `initialBalance` | 数値 | 開始時点の残高（円） |
| `annualContribution` | 数値 | 年間積立額（円） |
| `annualYieldPercent` | 数値 | 年利回り（%） |
| `contributionRaiseAmount` | 数値? | 積立額の年間増額 |
| `stopContributionYear` | 整数? | 積立停止年 |

**口座種別による違い:**

| 項目 | NISA | 課税口座 |
|---|---|---|
| 利益への課税 | なし | 20.315% |
| 年間投資上限 | 360 万円（※制限はモデル上未強制） | なし |
| 生涯非課税限度 | 1,800 万円（※制限はモデル上未強制） | なし |

課税口座の計算:
```
利回り(税前) = prevBalance × annualYieldPercent / 100
税額         = max(0, 利回り(税前)) × 20.315%
利回り(税後) = 利回り(税前) - 税額
期末残高     = prevBalance + 利回り(税後) + 積立額
```

## 3. シミュレーション アルゴリズム

### 3.1 全体フロー

```
入力: Scenario
出力: SimulationOutput { rows: YearlyResult[], warnings: SimulationWarning[] }
```

シミュレーションは**純粋関数**。外部状態に依存せず、入力から決定論的に出力を生成する。

### 3.2 事前処理

1. **年次範囲の生成**: `startYear` から `startYear + (endAge - husbandAge)` まで（両端含む）
2. **インフレ係数テーブル**: `inflationFactor[i] = (1 + rate/100)^i` を事前計算
3. **ローン年次スケジュール**: 各 `Loan` の月次返済表を生成し年次集約
4. **モード判定**: `taxConfig.incomeInputMode` を確認

### 3.3 年次ループ

```
繰越初期値:
  cashSavings = initialCashSavings
  investmentBalance = investment.initialBalance (or Σ accounts)
  cashNegativeReported = false

各年 y, インデックス i = 0, 1, ...:

  ┌─────────────────────────────────────────────────────┐
  │ (1) 年齢更新                                        │
  │     husbandAge = initialConditions.husbandAge + i   │
  │     wifeAge    = initialConditions.wifeAge + i      │
  │     childAge   = y - child.birthYear                │
  ├─────────────────────────────────────────────────────┤
  │ (2) ローン返済 + 年末残高                             │
  │     住宅ローンの年末残高を先に取得                      │
  │     （住宅ローン控除の計算に使用）                      │
  ├─────────────────────────────────────────────────────┤
  │ (3) 収入（モード分岐）                                │
  │     【手取りモード】                                  │
  │       husbandIncome = base + raise × i              │
  │       wifeIncome = base (産休判定)                   │
  │     【額面モード】                                    │
  │       夫: 額面 → 社保 → 控除 → 所得税 → 住民税        │
  │           → 住宅ローン控除 → 手取り                   │
  │       妻: 額面 → 同上（扶養なし、ローン控除なし）       │
  │       + 児童手当                                     │
  │       + 退職金（退職年のみ）                           │
  ├─────────────────────────────────────────────────────┤
  │ (4) 支出                                            │
  │     a. カテゴリ支出 × インフレ係数                    │
  │     b. 子ども: 食費 + 雑費 + 教育費 (× インフレ)     │
  │     c. ローン返済: 全ローンの当年合計                 │
  │     d. 固定資産税（額面モードのみ）                    │
  │     totalExpense = a + b + c + d                    │
  ├─────────────────────────────────────────────────────┤
  │ (5) 投資                                            │
  │     【単一口座】 yield → contribution → balance      │
  │     【複数口座】 各口座ごとに計算（課税口座は税引後）    │
  ├─────────────────────────────────────────────────────┤
  │ (6) キャッシュ更新                                    │
  │     cashSavings += totalIncome - totalExpense        │
  │                   - investmentContribution           │
  ├─────────────────────────────────────────────────────┤
  │ (7) 純資産                                           │
  │     netAssets = cash + invest - loanOutstanding      │
  ├─────────────────────────────────────────────────────┤
  │ (8) 警告判定                                         │
  │     cashSavings < 0 が初発 → warning 記録            │
  └─────────────────────────────────────────────────────┘
```

### 3.4 出力

#### YearlyResult（各年の結果レコード）

| フィールド | 説明 |
|---|---|
| `year` | 暦年 |
| `index` | 0-based インデックス |
| `husbandAge`, `wifeAge` | 夫・妻の年齢 |
| `children` | 各子の `{ id, name, age, stage }` |
| `husbandIncome`, `wifeIncome`, `totalIncome` | 収入（手取り。額面モード時は児童手当・退職金を含む） |
| `taxBreakdown` | 税内訳（額面モード時のみ、下表参照） |
| `expenseByCategory` | カテゴリ別支出 |
| `childExpense` | 子ども費用合計 |
| `childExpenseBreakdown` | `{ food, misc, education }` |
| `loanRepayment` | ローン返済合計（元利） |
| `loanInterestPaid` | 利息分 |
| `loanPrincipalPaid` | 元金分 |

### 3.5 恒等式（不変条件）

以下は全ての年で成立する:

```

```

テストでも全行についてこの整合性を検証している。

## 6. JSON スキーマ仕様

### 6.1 バージョニング

- `version: 1` または `version: 2` を受け付ける。
- v1 はインポート時に `migrateScenario()` で v2 に変換される。
- マイグレーション: `taxConfig`（takeHome モード）と `annualGross`（0）を追加。
- v1 シナリオを手取りモードで実行した場合、出力は v2 以前と完全一致する。

### 6.2 バリデーションルール

| フィールド | 制約 |
|---|---|
| `version` | `1` or `2` |
| `id` | 文字列 |
| `name` | 文字列 |
| `startYear` | 1900〜2200 の整数 |
| `husbandAge`, `wifeAge`, `endAge` | 0〜120 の整数 |
| `inflationRatePercent` | -20〜20 |
| `annualTakeHome` (夫・妻) | 0 以上 |
| `annualGross` (夫・妻) | 0 以上（任意） |
| `annualRaiseAmount` | 数値 |
| `maternityLeaveYears` | 0〜10 の整数 |
| 支出カテゴリ | 全 8 キー存在、`annualAmount >= 0`、`applyInflation: boolean` |
| `annualYieldPercent` | -50〜50 |
| `annualInterestRatePercent` | 0〜50 |
| `termYears` | 0〜60 の整数 |
| `educationPlan[stage]` | `'public'` または `'private'` |
| `educationCostOverride[stage]` | 0 以上（任意） |
| `childFoodAnnual`, `childMiscAnnual` | 0 以上 |
| `taxConfig` | 任意。存在する場合は各フィールドの型・値チェック |
| `investmentAccounts` | 任意。存在する場合は各要素の型・値チェック |

バリデーション失敗時は `ValidationError[]` を返す。
