### 2.1 レイヤー分離の方針

```
  types/     純粋な型定義のみ。ランタイムコードを含まない。
    ↓
  domain/    純粋関数。React・DOM に一切依存しない。
    ↓         テスト容易性の要。
  presets/   定数・ファクトリ関数。domain/ 層を参照可。
    ↓
  io/        JSON 入出力・バリデーション。DOM API (Blob等) を使用可。
    ↓
  state/     React の Context + useReducer。domain/ を参照可。
    ↓
  hooks/     カスタムフック。state/ と domain/ を橋渡し。
    ↓
  components/ React コンポーネント。上記すべてを参照可。
```

## 3. 状態管理

### 3.1 アーキテクチャ

```
<ScenarioProvider>           ← Context + useReducer
  ├─ scenario: Scenario      ← 全状態を 1 オブジェクトで管理
  ├─ dispatch: Dispatch<ScenarioAction>
  └─ hadRestored: boolean    ← localStorage 復元フラグ
```

- `useReducer` で `Scenario` を直接管理する。外部の状態管理ライブラリは使わない。
- `ScenarioProvider` はアプリのルート（`App.tsx` 内）に 1 つだけ配置。
- 全フォーム・テーブル・グラフは `useScenario()` で state を取得。

### 3.2 Reducer アクション

| アクション | payload | 説明 |
|---|---|---|
| `SET_SCENARIO` | `{ scenario }` | シナリオ全体の置換 |
| `RESET` | — | no-op（将来拡張用） |
| `SET_NAME` | `{ name }` | シナリオ名の変更 |
| `UPDATE_INITIAL` | `{ patch }` | `initialConditions` の部分更新 |
| `UPDATE_HUSBAND` | `{ patch }` | `income.husband` の部分更新 |
| `UPDATE_WIFE` | `{ patch }` | `income.wife` の部分更新 |
| `UPDATE_EXPENSE_CATEGORY` | `{ key, patch }` | 支出カテゴリの部分更新 |
| `UPDATE_INVESTMENT` | `{ patch }` | 投資設定の部分更新（単一口座） |
| `ADD_LOAN` | `{ loan }` | ローン追加 |
| `UPDATE_LOAN` | `{ id, patch }` | ローン更新 |
| `REMOVE_LOAN` | `{ id }` | ローン削除 |
| `ADD_CHILD` | `{ child }` | 子ども追加 |
| `UPDATE_CHILD` | `{ id, patch }` | 子ども更新 |
| `REMOVE_CHILD` | `{ id }` | 子ども削除 |
| `SET_INCOME_INPUT_MODE` | `{ mode }` | 収入入力モードの切替（v2） |
| `UPDATE_MORTGAGE_DEDUCTION` | `{ patch }` | 住宅ローン控除設定の更新（v2） |
| `UPDATE_PROPERTY_TAX` | `{ patch }` | 固定資産税設定の更新（v2） |
| `UPDATE_RETIREMENT_BONUS` | `{ patch }` | 退職金設定の更新（v2） |
| `ADD_INVESTMENT_ACCOUNT` | `{ account }` | 投資口座の追加（v2） |
| `UPDATE_INVESTMENT_ACCOUNT` | `{ id, patch }` | 投資口座の更新（v2） |
| `REMOVE_INVESTMENT_ACCOUNT` | `{ id }` | 投資口座の削除（v2） |

- `SET_SCENARIO` 以外のアクションは `updatedAt` を現在時刻に自動更新。
- Reducer は immutable 更新（スプレッド構文）で新しい `Scenario` を返す。
- v2 税金アクションは `taxConfig` が未定義の場合、デフォルト値で初期化してから更新する。

### 3.3 useSimulation

```typescript
function useSimulation(scenario: Scenario): SimulationOutput {
  return useMemo(() => simulate(scenario), [scenario]);
}
```

- `scenario` 参照が変わらなければキャッシュを返す。
- `simulate()` は純粋関数なので `useMemo` によるメモ化が安全。

## 4. データ永続化

### 4.1 localStorage 自動保存

| 項目 | 仕様 |
|---|---|
| キー | `life-plan:last-scenario` |
| 保存タイミング | `scenario` state が変化するたびに `useEffect` で保存 |
| 復元タイミング | 初回マウント時に `getItem` → `migrate()` → `validate()` → 成功なら復元 |
| 復元失敗時 | デフォルトテンプレートで開始 |
| UI 表示 | 復元成功時にヘッダに「編集中（ローカル保存あり）」バッジ |
| クリア | 「新規」ボタンで `localStorage.removeItem` + デフォルト |
| 実装 | `src/hooks/useLocalStoragePersist.ts` |

- 保存データは `JSON.stringify(scenario)` をそのまま格納。
- quota 超過時は `catch` で無視（データロストは許容）。
- v1 形式のデータが保存されていた場合、復元時に自動で v2 へマイグレーションされる。

### 4.2 JSON エクスポート

| 項目 | 仕様 |
|---|---|
| 形式 | JSON（`JSON.stringify(scenario, null, 2)` でインデント付き） |
| MIME | `application/json` |
| ファイル名 | `<シナリオ名>-<yyyyMMddHHmm>.json` |
| 方法 | `Blob` → `URL.createObjectURL` → `<a download>` |
| 実装 | `src/io/exportScenario.ts` |

### 4.3 JSON インポート

| 項目 | 仕様 |
|---|---|
| UI | `<input type="file" accept="application/json">` |
| 処理順 | `File.text()` → `JSON.parse` → `migrate()` → `validate()` → dispatch |
| エラー時 | ヘッダ下部にエラーメッセージ表示（先頭 3 件） |
| 実装 | `src/io/importScenario.ts` |

### 4.4 サンプル読込

- `public/scenarios/sample-default.json` を `fetch()` で取得。
- Vite の `public` ディレクトリから静的配信される。
- ヘッダの「サンプル読込」ボタンで発火。
- サンプルは v1 形式のため、インポート時に自動で v2 にマイグレーションされる。

### 4.5 バリデーション

- **zod は使用せず手書き**で必須キー・型・範囲チェックを行う。
- 検査項目の詳細はドメイン仕様書「6. JSON スキーマ仕様」を参照。
- 戻り値: `{ ok: boolean; errors: ValidationError[]; data?: Scenario }`
- v2 追加項目（`taxConfig`, `investmentAccounts`）は存在する場合のみバリデーション。
- 実装: `src/io/validate.ts`

### 4.6 マイグレーション

- `version` フィールドに応じて最新形式へ変換する。
- v1 → v2: `taxConfig`（takeHome モード）と `annualGross`（0）を追加。
- v2 シナリオはそのまま通過。
- 実装: `src/io/migrate.ts`

- **CSS カスタムプロパティ** (`styles/variables.css`) でテーマカラー・フォント・半径などを一元管理。

| `yen(value)` | 円表記 | `¥5,000,000` |
| `manYen(value)` | 万円単位（テーブル用） | `500万` |
| `numberJP(value)` | 通貨記号なし | `5,000,000` |
| `percent(value)` | 小数 1 桁のパーセント | `4.0%` |

### 7.2 ID 生成（`presets/defaultScenario.ts`）

```typescript
function cryptoRandomId(): string
```

- `crypto.randomUUID()` を優先。
- 未対応環境では `Math.random` + `Date.now` のフォールバック。
